## Q280126: Device Manager Indicates PCI Conflicts in Windows 95

	Article: Q280126
	Product(s): Microsoft Windows 95.x Retail Product
	Version(s): 
	Operating System(s): 
	Keyword(s): kb3rdparty kbhw kbHardware
	Last Modified: 27-JUL-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows 95 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you run Windows 95 on the following computers that have a PCI network
	adapter, you may see an exclamation point or a red "X" next to the PCI entry in
	Device Manager:
	
	- Netvista A40
	- Netvista A40P
	
	The model numbers include:
	
	- 6578
	- 6579
	- 6648
	- 6649
	- 6568
	- 6569
	
	CAUSE
	=====
	
	This behavior occurs because the hardware is certified to run on Windows
	98-based computers or later.
	
	RESOLUTION
	==========
	
	To resolve this behavior, contact IBM for more information.
	
	MORE INFORMATION
	================
	
	This behavior does not occur on Windows 98-based computers or later. The
	hardware that is listed in the "Symptoms" section has not been tested or
	certified to run on Windows 95-based computers and has architectural changes
	that are likely causing the behavior. This conflict does not appear to affect
	the functionality of the device but individual results may vary.

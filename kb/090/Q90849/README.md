## Q90849: Write: Saving in MS-DOS Word Format Loses Fonts and Graphics

	Article: Q90849
	Product(s): Microsoft Windows 95.x Retail Product
	Version(s): WINDOWS:3.0,3.0a,3.1,3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 29-OCT-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows versions 3.0, 3.0a, 3.1, 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	In Microsoft Windows Write, if you save a document in Microsoft Word for MS-DOS
	format, fonts, formatting, and any graphics are lost. This problem occurs even
	if you convert the file back to normal Write format (.WRI).
	
	This problem also occurs if you save a Write document as text. Spacing, margins,
	tabs, and fonts are not saved in a text file.
	
	If you create a Write document and import graphics into it, a message indicating
	that all graphics information cannot be saved is displayed.

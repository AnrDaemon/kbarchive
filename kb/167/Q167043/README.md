## Q167043: XCLN: NextExists and PreviousExists Properties Always True

{% raw %}

	Article: Q167043
	Product(s): Microsoft Exchange
	Version(s): WINDOWS:4.0,5.0; Win95:4.0,5.0
	Operating System(s): 
	Keyword(s): kbusage
	Last Modified: 08-JAN-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Windows 3.x client, versions 4.0, 5.0 
	- Microsoft Exchange Client for Windows 95, versions 4.0, 5.0 
	- Microsoft Exchange Windows NT client, versions 4.0, 5.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	
	When you are accessing the value of the NextExists or PreviousExists property of
	the MAPIFrom control in a Forms Designer form for Microsoft Exchange Server, the
	value returned is always true. Subsequently, a true value is returned even if a
	previous or next message does not exist.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in of Microsoft Exchange Server
	version 4.0. We are researching this problem and will post new information here
	in the Microsoft Knowledge Base as it becomes available.

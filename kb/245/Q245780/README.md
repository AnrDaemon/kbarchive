## Q245780: Access Violation in DNSAdmin.exe When Editing a Subnetted Revers

	Article: Q245780
	Product(s): Microsoft Windows NT
	Version(s): winnt:4.0
	Operating System(s): 
	Keyword(s): kbtool
	Last Modified: 08-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you use Windows NT Server Domain Name Server Administrator to edit a
	subnetted reverse lookup zone, you may receive an Access Violation error
	message. You may also notice when editing PTR records that the edit form appears
	to be corrupted.
	
	STATUS
	======
	
	Microsoft is researching this problem and will post more information here in the
	Microsoft Knowledge Base when it becomes available.

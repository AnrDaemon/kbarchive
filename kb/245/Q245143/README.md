## Q245143: MapPoint: Dialog Box Without Text Appears on the Screen

	Article: Q245143
	Product(s): Microsoft Automap
	Version(s): WINDOWS:; :
	Operating System(s): 
	Keyword(s): kberrmsg kbimu
	Last Modified: 14-SEP-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft MapPoint 2000 
	- Microsoft MapPoint 2001 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you attempt to quit Microsoft MapPoint, a dialog box containing no text may
	appear on the screen.
	
	CAUSE
	=====
	
	This behavior can occur if you attempt to quit MapPoint without saving a map you
	created in the Data Mapping wizard.
	
	RESOLUTION
	==========
	
	To work around this issue, click Yes to save the map.
	
	If you do not want to save the map, click No to quit MapPoint without saving the
	map.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in the Microsoft products that are
	listed at the beginning of this article.
	
	Additional query words: mp2000 mp2001 unsaved exit close

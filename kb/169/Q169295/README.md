## Q169295: Administration of Legacy RAS Servers from Routing and RAS Admin

	Article: Q169295
	Product(s): Microsoft Windows NT
	Version(s): WinNT:4.0
	Operating System(s): 
	Keyword(s): kbnetwork
	Last Modified: 09-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 4.0 
	- Microsoft Routing and Remote Access Service Update for Windows NT Server version 4.0 
	-------------------------------------------------------------------------------
	
	SUMMARY
	=======
	
	It is possible to remotely administer a downlevel Remote Access Services (RAS)
	server using the Routing and RAS Administrator tool.
	
	MORE INFORMATION
	================
	
	To administer a downlevel RAS server using a Routing and Remote Access router,
	designate the downlevel server in Routing and RAS Administrator. Start Routing
	and RAS Administrator From the Server menu, click Connect to Router. In the
	Connect to Router dialog box type the computer name or IP address of the
	downlevel RAS server. This brings up the downlevel Remote Access Administrator
	application. This action requires administrative privileges on the remote RAS
	server.

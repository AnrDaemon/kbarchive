## Q194949: XADM: The Default Setting for Canceling Schema is Wrong

	Article: Q194949
	Product(s): Microsoft Exchange
	Version(s): WinNT:5.5
	Operating System(s): 
	Keyword(s): 
	Last Modified: 08-MAY-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 5.5 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you run the Move Server Wizard, a pop-up message might appear indicating
	"Loading schema". If you click Cancel, a Yes/No pop-up dialog box appears with
	the default set to Yes. All defaults in User Interface pages for the Move Server
	Wizard are set to be default equal to "No," except for this one.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Exchange Server
	version 5.5.
	
	
	Additional query words: Pilgrim, Loading Schema

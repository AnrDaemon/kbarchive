## Q135622: DOC: Extra Characters After Printing String Comparision

	Article: Q135622
	Product(s): Microsoft FoxPro
	Version(s): WINDOWS:3.0
	Operating System(s): 
	Keyword(s): kbdocfix
	Last Modified: 15-FEB-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Visual FoxPro for Windows, version 3.0 
	-------------------------------------------------------------------------------
	
	SUMMARY
	=======
	
	In the Developer's Guide on Page 458, a greater than sign ('>') followed by a
	less than sign ('<') appear on the line following the code example for "=="
	character string comparison operator, under the "Comparing Strings" heading.
	These characters are extra and should not appear.
	
	This documentation error has been fixed in Visual FoxPro 5.0 and 3.0b for
	Windows.
	
	Additional query words: 3.00 docerr VFoxWin 3.00b 5.00

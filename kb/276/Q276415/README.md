## Q276415: Slow Performance When Lotus Notes Is Running on Terminal Server

	Article: Q276415
	Product(s): Microsoft Windows NT
	Version(s): 4.0
	Operating System(s): 
	Keyword(s): kb3rdparty kbnetwork
	Last Modified: 08-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 4.0, Terminal Server Edition 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	On a terminal server that is running Lotus Notes 5.0, there may be a delay in
	the appearance of typed data. There may be a delay of up to 10 words before the
	words that the user has typed appear on the screen.
	
	RESOLUTION
	==========
	
	To resolve this issue, upgrade to Lotus Notes Client release 5.02.
	
	MORE INFORMATION
	================
	
	The third-party products discussed in this article are manufactured by vendors
	independent of Microsoft; we make no warranty, implied or otherwise, regarding
	these products' performance or reliability.
	
	Lotus Notes Client release 5.0.2 (and later R5 releases) is supported by Notes
	Support Services, but not certified when it is deployed on Microsoft Windows NT
	Server 4.0, Terminal Server Edition. Lotus Notes Client release 5.0.3 (and later
	R5 releases) is supported by Notes Support Services, but not certified when it
	is deployed on Microsoft Windows 2000 Terminal Services. Lotus supports Lotus
	Notes version 5.02 on a terminal server, but not Lotus Notes version 5.0.

## Q85011: Character Map Causes Windows Not to Tile Correctly

	Article: Q85011
	Product(s): Microsoft Windows 95.x Retail Product
	Version(s): WINDOWS:3.1,3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 09-NOV-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows versions 3.1, 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	Microsoft Windows version 3.1 Character Map, CHARMAP.EXE, causes awkward screen
	layout if it is one of the current applications running (not minimized) when
	Task List's Tile command is used. This behavior is caused by the fixed-size
	window that CHARMAP.EXE uses. This problem also effect applications like Control
	Panel and Calculator in scientific mode.
	
	WORKAROUND
	==========
	
	To work around this problem:
	
	1. Do not use the Task List's Tile command.
	
	2. Manually resize and move the programs.
	
	NOTE: You cannot resize Character Map.
	
	Additional query words: calc 3.10 3.11

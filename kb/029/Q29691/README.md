## Q29691: Conditions Where /PACKCODE is Not the Default

{% raw %}

	Article: Q29691
	Product(s): Microsoft Programming Utilities
	Version(s): MS-DOS:5.01,5.02,5.03,5.05,5.1,5.11,5.13,5.15,5.2,5.3
	Operating System(s): 
	Keyword(s): kb16bitonly
	Last Modified: 04-NOV-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft LINK for MS-DOS, versions 5.01, 5.02, 5.03, 5.05, 5.1, 5.11, 5.13, 5.15, 5.2, 5.3 
	-------------------------------------------------------------------------------
	
	SUMMARY
	=======
	
	/PACKCODE is normally the default when linking. However, if either of the
	following is true, /NOPACKCODE is the default:
	
	1. Segments are defined in the .DEF file
	
	-or-
	
	2. The /INCR[EMENTAL] switch is given

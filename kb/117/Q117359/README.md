## Q117359: SFM: Copying Files Between Volumes Freezes Other Mac Clients

	Article: Q117359
	Product(s): Microsoft Windows NT
	Version(s): 3.1 3.5
	Operating System(s): 
	Keyword(s): kbnetwork
	Last Modified: 08-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Advanced Server, version 3.1 
	- Microsoft Windows NT Server version 3.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When a Macintosh client copies a file from a shared disk volume on a Windows NT
	server running Services for Macintosh (SFM) to another volume on the same
	server, other Macintosh clients that have connections to the volume freeze,
	showing network activity, until the file transfer is complete.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT and Windows NT
	Advanced Server version 3.1 and Windows NT Workstation and Windows NT Server
	version 3.5. This problem was corrected in the latest U.S. Service Pack for
	Windows NT version 3.5. For information on obtaining the Service Pack, query on
	the following word in the Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K
	
	Additional query words: prodnt sfm

## Q179617: Explorer Hangs with Multiple File Operations in a Single Folder

{% raw %}

	Article: Q179617
	Product(s): Microsoft Windows NT
	Version(s): WinNT:4.0
	Operating System(s): 
	Keyword(s): 
	Last Modified: 09-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 4.0 
	- Microsoft Windows NT Workstation version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	Explorer is slow to respond when a file is renamed at the same time any other
	file in the same directory is opened or closed by another process.
	
	Explorer will seem to stop responding (hang) for about 30 seconds before
	returning to its initial state.
	
	STATUS
	======
	
	Microsoft is researching this problem and will post new information here in the
	Microsoft Knowledge Base as it becomes available.
	
	
	Additional query words: freeze hangs hung not responding explore

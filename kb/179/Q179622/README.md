## Q179622: Err Msg: Encarta Was Not Able to Integrate Any New Updates

{% raw %}

	Article: Q179622
	Product(s): Microsoft Home Multimedia Titles
	Version(s): WINDOWS:
	Operating System(s): 
	Keyword(s): kberrmsg kbimukbfaq
	Last Modified: 07-MAY-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Encarta 98 Encyclopedia for Windows 
	- Microsoft Encarta Encyclopedia 99 
	- Microsoft Encarta Reference Suite 99 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you double-click an Encarta Monthly Update file, you may receive the
	following error message:
	
	  Encarta was not able to integrate any new updates.
	
	CAUSE
	=====
	
	This behavior can occur if you download the Monthly Update file, and then save
	the update file to any folder other than the default folder for update files.
	
	RESOLUTION
	==========
	
	To resolve this behavior, move the Monthly Update file to the default folder for
	update files.
	
	The location of the default folder for update files in Encarta 98 is:
	
	  Program Files\Microsoft Reference\Encarta Encyclopedia\Update98
	
	The location of the default folder for update files in Encarta 99 is:
	
	  Program Files\Microsoft Reference\Encarta Encyclopedia\Update99
	
	To integrate the Monthly Update file contents into Encarta, double-click the
	update file after you move it to the appropriate default folder.
	
	MORE INFORMATION
	================
	
	For information about how to move a file in Windows, see the "Moving a File or
	Folder" Help topic in Microsoft Windows 95 or the "Moving Files, Folders" Help
	topic in Microsoft Windows 98.
	
	To view this topic, click Start, click Help, click the Index tab, type "moving"
	(without the quotation marks), click the "Files or Folders" or "Moving Files,
	Folders" Help topic, and then click Display.

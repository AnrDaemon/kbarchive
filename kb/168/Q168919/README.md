## Q168919: Windows NT Explorer Does Not Refresh New DFS Mappings

{% raw %}

	Article: Q168919
	Product(s): Microsoft Windows NT
	Version(s): 4.0
	Operating System(s): 
	Keyword(s): kbnetworkkbfixlist
	Last Modified: 25-FEB-2002
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Workstation version 4.0 
	- Microsoft Windows NT Server version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you use Windows NT Explorer to map a new drive letter to the root of a
	Distributed File System (Dfs) tree, Windows NT Explorer does not show the new
	drive letter until the view is refreshed either from the View\Refresh menu
	option or by pressing the F5 key.
	
	
	RESOLUTION
	==========
	
	At this time, the only resolution for this problem is a manual refresh of the
	Windows NT Explorer window.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT version 4.0. We are
	researching this problem and will post new information here in the Microsoft
	Knowledge Base as it becomes available.
	
	Additional query words: dfs explorer refresh

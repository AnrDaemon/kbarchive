## Q168088: NTMMTA Leaves Os2.exe in Memory When Run as Service on NT 4.0

	Article: Q168088
	Product(s): Microsoft Mail For PC Networks
	Version(s): WINDOWS:3.5
	Operating System(s): 
	Keyword(s): 
	Last Modified: 05-NOV-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Mail for PC Networks, version 3.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When the Mail Multitasking MTA for Windows NT (NTMMTA) is configured to run as a
	service on Windows NT 4.0, after the NTMMTA service is stopped, stranded
	instances of Os2.exe will still be running.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Mail for PC Networks,
	version 3.5. Microsoft is researching this problem and will post new information
	here in the Microsoft Knowledge Base as it becomes available.
	
	Additional query words: ntmmta nt4 os2.exe

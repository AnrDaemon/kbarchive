## Q88584: Blank Page Prints When Using Windows 3.1 PostScript Driver

	Article: Q88584
	Product(s): Microsoft Windows 95.x Retail Product
	Version(s): WINDOWS:3.1,3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 01-NOV-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows versions 3.1, 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	If you are using the Windows 3.1 PostScript printer driver version 3.5 or 3.53,
	and select both the Negative Image check box and the Conform To Adobe Document
	Structuring Conventions check box, you get a blank page when you print from any
	Windows-based application. If you clear either of these check boxes, you will be
	able to print normally.
	
	WORKAROUND
	==========
	
	To work around this problem:
	
	1. Run Control Panel.
	
	2. Choose the Printer icon.
	
	3. Select the PostScript printer from the list of installed printers.
	
	4. Choose the Setup button.
	
	5. Choose the Options button.
	
	6. Choose the Advanced button.
	
	7. Clear either the Negative Image check box or the Conform To Adobe Document
	  Structuring Conventions check box.
	
	8. Choose the next three OK buttons to save your changes.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows version 3.1. We are
	researching this problem and will post new information here as it becomes
	available.

## Q137317: Bookshelf/Encarta: Wrong Fonts Are Displayed

	Article: Q137317
	Product(s): Microsoft Home Multimedia Titles
	Version(s): WINDOWS:3.1,3.11; :1995 edition
	Operating System(s): 
	Keyword(s): 
	Last Modified: 13-JUN-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Bookshelf 1996-97 for Windows 
	- Microsoft Bookshelf for Windows 1995 edition 
	- Microsoft Encarta 95 The Complete Interactive Multimedia Encyclopedia for Macintosh 
	- Microsoft Encarta 97 Encyclopedia for Windows 
	- Microsoft Encarta Encyclopedia 97 Deluxe for Windows 
	- the operating system: Microsoft Windows NT, versions 3.1, 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you use one of the multimedia programs listed at the top of this article,
	text appears in the wrong font. This occurs after the program has previously run
	correctly.
	
	With Encarta 1997, the following error message is displayed after starting the
	program:
	
	Encarta Encyclopedia cannot find one or more of the fonts it uses. Encarta
	Encyclopedia will run without these fonts, but some text may not display
	properly. To correct the problem, run Setup again from the CD-ROM.
	
	CAUSE
	=====
	
	The problem occurs if Windows TrueType fonts are not active (disabled).
	
	If TrueType fonts are deactivated after the multimedia application is installed,
	a substitute bitmap font is used. Depending upon the system, the substituted
	font can be random.
	
	RESOLUTION
	==========
	
	Activate (enable) TrueType fonts and restart Windows. For more information about
	how to accomplish this task in Windows, see your Windows printed documentation
	or online help.
	
	MORE INFORMATION
	================
	
	The installation of Encarta or Bookshelf requires activating TrueType fonts. If
	TrueType fonts are not activated, you are prompted during Setup to activate
	them.

## Q173222: WD97: Line Between Columns Fails to Appear and Is Not Printed

	Article: Q173222
	Product(s): Word 97 for Windows
	Version(s): WINDOWS:97
	Operating System(s): 
	Keyword(s): kbdta winword word97 kblayout
	Last Modified: 14-NOV-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Word 97 for Windows 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you format a section in a document for multiple columns with the Line
	between check box selected (on the Format menu, click Columns) and then you do
	the following:
	
	- Insert a next page section break
	
	  -and-
	
	- Format the following section in landscape orientation,
	
	the line between the columns in the previous section does not appear on the
	screen or print on the last page of the multiple column section.
	
	CAUSE
	=====
	
	This problem is related to the landscape orientation of the section following
	the section that is formatted for multiple columns with the Line between check
	box selected.
	
	WORKAROUND
	==========
	
	You can work around the problem by using one of the following methods.
	
	Method 1: Insert an Additional Break
	------------------------------------
	
	Insert an additional continuous section break before the next page section break
	(the one that starts the landscape section). To do this, click Break on the
	Insert menu, and select Continuous.
	
	Method 2: Draw a Line Using the Drawing Tools
	---------------------------------------------
	
	You can turn on the Drawing toolbar and use the Line tool to draw a vertical
	black line between the columns. After you draw the vertical line, you can adjust
	the formatting for the line by following these steps:
	
	1. Right-click the vertical line and click Format AutoShape.
	
	2. Click the Colors and Lines tab.
	
	3. Adjust the Style and Weight boxes to achieve the result you want.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in the Microsoft products that are
	listed at the beginning of this article.

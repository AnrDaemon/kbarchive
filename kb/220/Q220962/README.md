## Q220962: XWEB: Adding Non-Existent Attachment in OWA Creates New File

{% raw %}

	Article: Q220962
	Product(s): Microsoft Exchange
	Version(s): WINDOWS:5.5
	Operating System(s): 
	Keyword(s): 
	Last Modified: 22-JUN-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Outlook Web Access, version 5.5 
	- Microsoft Outlook Web Access, version 5.5 Service Packs 1, 2, 3 
	- Microsoft Exchange Server, version 5.5 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you use the Outlook Web Access (OWA) client to access your mailbox, you can
	send messages with attachments just as with a standard mail client. However,
	when you select the Attachments tab and type the name of a non-existent file,
	clicking Add Attachment Now will create and attach a blank file. Clicking on the
	attachment will open an empty Notepad file.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in the Microsoft products that are
	listed at the beginning of this article.

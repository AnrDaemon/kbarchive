## Q218987: XCLN: Attachment is Empty if Pasted Twice to Desktop

{% raw %}

	Article: Q218987
	Product(s): Microsoft Exchange
	Version(s): 8.0,8.01,8.02,8.03; WINDOWS:
	Operating System(s): 
	Keyword(s): 
	Last Modified: 22-JUN-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Outlook 97, versions 8.0, 8.01, 8.02, 8.03 
	- Microsoft Outlook 98 
	- Microsoft Exchange Server, version 5.5 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you open a mail message from within Outlook with an attachment and paste
	the attachment to your desktop, you may lose data. This will happen if you copy
	the attachment and paste the attachment two times. The second attachment will be
	empty. The data is missing from the second attachment, or if the second
	attachment overwrites the first attachment, it will be empty.
	
	WORKAROUND
	==========
	
	After copying and pasting the attachment the first time, recopy the attachment,
	and paste again. The contents of the attachment will still be there.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in the Microsoft products that are
	listed at the beginning of this article.

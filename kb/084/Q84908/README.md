## Q84908: Calculator Does Not Display Zeroes When Entering Decimals

	Article: Q84908
	Product(s): Microsoft Windows 95.x Retail Product
	Version(s): WINDOWS:3.1,3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 14-DEC-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows versions 3.1, 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When a decimal number, such as 10.0005, is entered in Microsoft Windows
	Calculator (CALC.EXE) it displays only the first zero after the decimal point
	until a number other than zero is entered.
	
	The Windows 3.1 Calculator behaves this way in both its scientific and standard
	modes of operation.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows version 3.1. We are
	researching this problem and will post new information here as it becomes
	available.

## Q164262: Windows Explorer Refresh Changes Current Selection

	Article: Q164262
	Product(s): Microsoft Windows NT
	Version(s): WINNT:4.0
	Operating System(s): 
	Keyword(s): kbui
	Last Modified: 09-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Workstation version 4.0 
	- Microsoft Windows NT Server version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	After a refresh in the Windows NT Explorer, the previously selected object is
	not selected anymore. After the refresh, the first item in the current Explorer
	window has the focus, but it is not selected.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT version 4.0. We are
	researching this problem and will post new information here in the Microsoft
	Knowledge Base as it becomes available.
	
	MORE INFORMATION
	================
	
	This problem occurs if you refresh the window by using either the Refresh
	command on the View menu, or the F5 key.

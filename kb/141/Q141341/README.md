## Q141341: XCLN: No Print Option When Using EFD on Windows NT

{% raw %}

	Article: Q141341
	Product(s): Microsoft Exchange
	Version(s): 4.0
	Operating System(s): 
	Keyword(s): kbprint kbPrinting
	Last Modified: 19-FEB-2002
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 4.0 
	-------------------------------------------------------------------------------
	
	SUMMARY
	=======
	
	Microsoft Exchange Electronic Forms Designer version 4.0 does not have an option
	for Print Setup when you run on Microsoft Windows NT. When you print from a
	Forms Designer Electronic Form, the form will use the Windows NT system printing
	defaults.

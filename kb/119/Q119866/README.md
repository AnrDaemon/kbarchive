## Q119866: Microcom 4232 Modem Connects at 9600 Baud Only

	Article: Q119866
	Product(s): Microsoft Windows NT
	Version(s): 3.10 3.50
	Operating System(s): 
	Keyword(s): 
	Last Modified: 08-AUG-2001
	
	3.10 3.50
	
	WINDOWS
	
	kbnetwork kb3rdparty kbbug3.10 kbbug3.50
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 3.1 
	- Microsoft Windows NT Workstation version 3.1 
	- Microsoft Windows NT Advanced Server, version 3.1 
	- Microsoft Windows NT Workstation version 3.5 
	- Microsoft Windows NT Server version 3.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you use Remote Access Service (RAS) with a Microcom 4232 v.32 bis modem
	connected to a Digiboard PC/16XEM, client machines, including computers with
	Windows for Workgroups version 3.11 installed, can connect only with the baud
	rate of 9600 baud.
	
	WORKAROUND
	==========
	
	You can work around this problem by doing one of the following:
	
	- Use one of the available built-in communication ports on your computer.
	
	  -or-
	
	- Use another supported modem connected to the Digiboard PC/16XEM.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT and Windows NT
	Advanced Server version 3.1 and Windows NT Workstation and Windows NT Server
	version 3.5. We are researching this problem and will post new information here
	in the Microsoft Knowledge Base as it becomes available.

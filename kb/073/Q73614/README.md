## Q73614: Video Corruption Can Occur in MS-DOS Shell During Swap

{% raw %}

	Article: Q73614
	Product(s): Microsoft Disk Operating System
	Version(s): MS-DOS:5.0
	Operating System(s): 
	Keyword(s): 
	Last Modified: 23-NOV-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft MS-DOS operating system version 5.0 
	-------------------------------------------------------------------------------
	
	SUMMARY
	=======
	
	With MS-DOS Shell Task Swapper enabled, moving the mouse at a fast pace while
	swapping back to an application can cause video corruption. The video may
	freeze, the machine may hang, the mouse may lose synchronization, or the mouse
	may become inactive.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft MS-DOS 5.0 and 6.0
	Shell. We are researching this problem and will post new information in the
	Microsoft Knowledge Base as it becomes available.

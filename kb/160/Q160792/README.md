## Q160792: &quot;Snap to default&quot; Mouse Feature Not Working

	Article: Q160792
	Product(s): Microsoft Windows NT
	Version(s): WinNT:3.5,3.51,4.0
	Operating System(s): 
	Keyword(s): kbenv kbinterop kbui
	Last Modified: 09-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Workstation versions 3.5, 3.51, 4.0 
	- Microsoft Windows NT Server versions 3.5, 3.51, 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	The "Snap to default" option found in the mouse configuration of Control Panel
	may not work in some applications in Windows NT.
	
	CAUSE
	=====
	
	Some applications create dialog boxes in an inconsistent manner, causing this
	feature to fail.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT version 3.5x and 4.0
	We are researching this problem and will post new information here in the
	Microsoft Knowledge Base as it becomes available.

## Q126580: SMS Windows 95 Client UNC Connections Not Dropped

	Article: Q126580
	Product(s): Microsoft Systems Management Server
	Version(s): winnt:1.1
	Operating System(s): 
	Keyword(s): kbnetwork kbPGC smspgc
	Last Modified: 30-JUL-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Systems Management Server version 1.1 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	On the Systems Management Server Windows 95 client, Universal Naming Convention
	(UNC) connections are not dropped immediately after an application using the UNC
	connection is closed. The connection can take several minutes to close.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Systems Management Server
	version 1.1. This problem has been corrected in Systems Management Server
	version 1.2.

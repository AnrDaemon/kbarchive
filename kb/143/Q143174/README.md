## Q143174: WSLAVELM.EXE (or WSLAVEIPX.EXE) Times Out After 20 Seconds

	Article: Q143174
	Product(s): Microsoft Systems Management Server
	Version(s): winnt:1.1
	Operating System(s): 
	Keyword(s): kbnetwork kbsmsUtil smsutil
	Last Modified: 31-JUL-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Systems Management Server version 1.1 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you choose File Transfer in Help Desk of the Systems Management Server
	Admin UI for a client computer and there is no network activity for more than 20
	seconds, WSLAVELM.EXE (or WSLAVEIPX.EXE) on the client computer times out.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Systems Management Server
	version 1.1. This problem has been corrected in Systems Management Server
	version 1.2.

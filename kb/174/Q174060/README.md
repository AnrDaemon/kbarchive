## Q174060: XADM: Storage Limits Cannot be Propagated to Subfolders

	Article: Q174060
	Product(s): Microsoft Exchange
	Version(s): 5.5
	Operating System(s): 
	Keyword(s): 
	Last Modified: 22-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 5.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	
	When you view the properties of a top level Public Folder (PF) in the Microsoft
	Exchange Administrator program, the option to propagate Microsoft Exchange
	Information Store (IS) storage limits to its subfolders does not exist.
	
	CAUSE
	=====
	
	The current version of Exchange Server does not allow the propagation of this
	setting from top level folders to subfolders.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be an issue in Microsoft Exchange Server,
	version 5.5. We are researching this issue and will post more information here
	in the Microsoft Knowledge Base as it becomes available.

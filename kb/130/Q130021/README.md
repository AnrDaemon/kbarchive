## Q130021: XCLN: Copy from To: Field Doesn't Copy Trailing Semicolon

	Article: Q130021
	Product(s): Microsoft Exchange
	Version(s): 4.0 5.0
	Operating System(s): 
	Keyword(s): kbusage
	Last Modified: 10-MAR-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Windows 3.x client, versions 4.0, 5.0 
	- Microsoft Exchange Windows NT client, versions 4.0, 5.0 
	- Microsoft Exchange Windows 95/98 client, versions 4.0, 5.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you copy the contents of a To, Cc, or Bcc field and paste the text outside
	the To, Cc, or Bcc field, any trailing semicolons are removed.
	
	If you copy a string that contains multiple names, all semicolons are copied
	except for the one (or more) at the end.
	
	CAUSE
	=====
	
	This is by product design.

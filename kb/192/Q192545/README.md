## Q192545: PC WRmt: Windows Remote Client Displays Wrong Date after 2000

{% raw %}

	Article: Q192545
	Product(s): Microsoft Mail For PC Networks
	Version(s): WINDOWS:3.2
	Operating System(s): 
	Keyword(s): 
	Last Modified: 23-JAN-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Mail Remote for Windows, version 3.2 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	
	After the year 2000, the Windows Remote client may display the year as 10 instead
	of 2000. The callin and callout logs may also reflect this date incorrectly.
	
	CAUSE
	=====
	
	The problem is caused by running an older version of the MSRemote program.
	
	RESOLUTION
	==========
	
	Upgrade these clients to version 3.2.2000.4025.

## Q153640: Run Local Copy If Present&quot; Removes Local Executable Files

	Article: Q153640
	Product(s): Microsoft Systems Management Server
	Version(s): winnt:1.0,1.1
	Operating System(s): 
	Keyword(s): kbnetwork kbPGC smspgc
	Last Modified: 30-JUL-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Systems Management Server versions 1.0, 1.1 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	
	When you launch a shared application configured for "Run Local Copy If Present,"
	Program Group Control (PGC) displays a dialog box which explains that continuing
	with the configuration overwrites the local copy with the shared copy.
	
	If you click NO, PGC still deletes the local executable files.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Systems Management Server
	versions 1.0 and 1.1. This problem has been corrected in Systems Management
	Server version 1.2.
	
	Additional query words: prodsms run local copy

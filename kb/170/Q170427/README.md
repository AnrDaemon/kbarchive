## Q170427: XFOR: Hebrew Zayin Character Not Preserved

{% raw %}

	Article: Q170427
	Product(s): Microsoft Exchange
	Version(s): winnt:5.0
	Operating System(s): 
	Keyword(s): exc5
	Last Modified: 10-FEB-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 5.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	Users and administrators of Microsoft Exchange Server version 5.0 will find the
	Hebrew zayin character is replaced by "ae" in messages sent from Microsoft Mail
	(MS Mail) clients to MS Mail clients connected through the Exchange Internet
	Mail Service. The message must go through a SMTP gateway connected to the
	recipient's postoffice in order for this problem to occur.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Exchange Server
	version 4.0. This problem was corrected in Microsoft Exchange Server 5.0 U.S.
	Service Pack 2. For information on obtaining the service pack, query on the
	following word in the Microsoft Knowledge Base (without the spaces): S E R V P A
	C K

## Q155540: Copying File Returns File Not Found Instead of Invalid Path

	Article: Q155540
	Product(s): Microsoft Windows NT
	Version(s): winnt:3.5,3.51
	Operating System(s): 
	Keyword(s): kbnetwork
	Last Modified: 08-AUG-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server versions 3.5, 3.51 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you use the File Copy command to copy a file to a File and Print Services
	for NetWare server, and the path you enter in the To box does not exist, you
	receive a "File Not Found" error rather than an "Invalid Path" error.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT version 3.51. This
	problem has been corrected in the latest Microsoft Windows NT U.S. Service Pack.
	For information on obtaining the Service Pack, query on the following word in
	the Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K
	
	
	NOTE: This article is not mentioned in the Windows NT 3.51 Service Pack 5
	Readme.txt file.
	
	Additional query words: prodnt

## Q155812: XADM: NDR from Japanese Microsoft Exchange Server is Corrupt

	Article: Q155812
	Product(s): Microsoft Exchange
	Version(s): winnt:4.0
	Operating System(s): 
	Keyword(s): kbusage
	Last Modified: 04-APR-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	In a Japanese environment, the Japanese version of the Microsoft Exchange
	Microsoft Mail Connector is used to connect to the Japanese version of Microsoft
	Mail 3.x. When you send messages through the Microsoft Mail Connector and
	address them to a valid Postoffice and an invalid mailbox, the Non-Delivery
	Report (NDR) that you receive will be partly corrupted.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Exchange Server
	version 4.0. This problem was corrected in the latest Microsoft Exchange Service
	Pack. For information on obtaining the Service Pack, query on the following word
	in the Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K
	
	
	Additional query words: DBCS JP JPN NDR

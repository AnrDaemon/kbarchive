## Q123358: When TCP/IP-32 Under Stress, MS-DOS Command Prompt GP Faults

{% raw %}

	Article: Q123358
	Product(s): Windows for Workgroups and Windows NT Networking Issues
	Version(s): 3.11; WINDOWS:3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 24-OCT-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft TCP/IP-32 for Windows for Workgroups, version 3.11 
	- Microsoft Windows for Workgroups version 3.11 
	-------------------------------------------------------------------------------
	
	
	SUMMARY
	=======
	
	If you open an MS-DOS command prompt during a period when TCP/IP-32 is under
	extreme stress, the command prompt window may terminate with a general
	protection (GP) fault.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in TCP/IP-32. This problem has been
	corrected in an updated TCP/IP-32 version 3.11a. For information on the
	availability of TCP/IP-32 3.11a, see article Q122544 in the Microsoft Knowledge
	Base.
	
	Additional query words: wfw wfwg prodtcp32 3.11 3.11a

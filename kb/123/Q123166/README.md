## Q123166: Time and Date Stamp of Word 6.0 Document Updated

	Article: Q123166
	Product(s): Microsoft Windows NT
	Version(s): 3.5,6.0,6.0.1,6.0.1a
	Operating System(s): 
	Keyword(s): 
	Last Modified: 08-AUG-2001
	
	3.50
	
	WINDOWS
	
	kbinterop kbbug3.50 kbfix3.50.sp2
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 3.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you close an unmodified Microsoft Word version 6.0 document located on a
	Windows NT Server version 3.5 from a Windows for Workgroups version 3.11 client
	computer, the time and date stamp of the document is updated.
	
	NOTE: This behavior does not occur with a Windows NT Workstation version 3.5 as a
	client computer.
	
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Windows NT version
	3.5. This problem was corrected in the latest U.S. Service Pack for Windows NT
	version 3.5. For information on obtaining the Service Pack, query on the
	following word in the Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K
	
	
	Additional query words: wfw wfwg prodnt winword word6

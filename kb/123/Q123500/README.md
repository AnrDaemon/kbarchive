## Q123500: Administrators Permissions for a Printer Change to Read

	Article: Q123500
	Product(s): Microsoft Windows NT
	Version(s): 
	Operating System(s): 
	Keyword(s): 
	Last Modified: 08-AUG-2001
	
	3.50
	
	WINDOWS
	
	kbsetup kbbug3.50 kbfix3.50.sp2
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Workstation version 3.5 
	- Microsoft Windows NT Server version 3.5 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	After you upgrade a Windows NT 3.1 installation to Windows NT version 3.5 and
	reboot the computer several times, the permission on
	HKEY_CURRENT_USERS\SOFTWARE\MICROSOFT\WINDOWSNT\DEVICE and
	HKEY_CURRENT_USERS\SOFTWARE\MICROSOFT\WINDOWSNT\CURRENT VERSION\DEVICE in change
	to read for the administrator.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT Workstation and
	Windows NT Server version 3.5. This problem was corrected in the latest U.S.
	Service Pack for Windows NT version 3.5. For information on obtaining the
	Service Pack, query on the following word in the Microsoft Knowledge Base
	(without the spaces):
	
	  S E R V P A C K
	
	Additional query words: prodnt

## Q242356: Remote Control Logging Does Not Work in SMS 2.0 SP1

	Article: Q242356
	Product(s): Microsoft Systems Management Server
	Version(s): winnt:2.0 SP1
	Operating System(s): 
	Keyword(s): kbsms200bug kbsms200fix kbsms200sp2fix
	Last Modified: 01-AUG-2000
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Systems Management Server version 2.0 SP1 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	In Systems Management Server (SMS) versions earlier than SMS 2.0 Service Pack 1
	(SP1), you can enable low-level remote control logging to help troubleshoot
	complex remote control problems. For additional information about this logging
	and how to enable it, click the article number below to view the article in the
	Microsoft Knowledge Base:
	
	  Q195859 SMS: How to Enable Remote Control Logging for WUSER in SMS 2.0
	
	In SMS 2.0 SP1, this logging does not work. The log file is created but nothing
	is written to it.
	
	RESOLUTION
	==========
	
	To resolve this problem, obtain the latest service pack for Systems Management
	Server version 2.0. For additional information, please see the following article
	in the Microsoft Knowledge Base:
	
	  Q236325 How to Obtain the Latest Systems Management Server 2.0 Service Pack
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Systems Management Server
	version 2.0. This problem was first corrected in Systems Management Server
	version 2.0 Service Pack 2.

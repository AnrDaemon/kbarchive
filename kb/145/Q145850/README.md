## Q145850: Streets 4.0: Map Not Redrawn When Dialog Box is Moved

	Article: Q145850
	Product(s): Microsoft Automap
	Version(s): WINDOWS:4.0
	Operating System(s): 
	Keyword(s): 
	Last Modified: 25-AUG-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Automap Streets for Windows, version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you move Automap Streets dialog boxes, a blank area appears on your screen
	in place of the dialog box you moved.
	
	The problem occurs in the following dialog boxes:
	
	- Home location settings.
	
	- Secondary dialog boxes generated when you click the "Layers" button, located
	  in the Options section of the Map menu.
	
	
	WORKAROUND
	==========
	
	In the dialog box, click OK or Cancel.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Automap Streets
	version 4.0 for Windows. We are researching this problem and will post new
	information here in the Microsoft Knowledge Base as it becomes available.

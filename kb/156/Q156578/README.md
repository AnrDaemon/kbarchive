## Q156578: Cannot Cancel Print Job on Windows NT 3.51 Shared Printer

{% raw %}

	Article: Q156578
	Product(s): Microsoft Windows NT
	Version(s): 4.0
	Operating System(s): 
	Keyword(s): kbnetwork kbprint kbPrinting
	Last Modified: 07-SEP-2001
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows NT Server version 4.0 
	- Microsoft Windows NT Workstation version 4.0 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	When you are printing from a Windows NT 4.0-based computer to a shared printer
	connected to a Windows NT 3.51-based computer, you may receive the following
	error message in the status bar if you try to cancel a print job in the Print
	Queue window:
	
	  Error processing command
	
	When this occurs, the print job is not deleted.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Windows NT version 4.0. This
	problem was corrected in the latest Windows NT 4.0 U.S. Service Pack. For
	information on obtaining the service pack, query on the following word in the
	Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K
	
	Additional query words: prodnt sp2 delete remotely administer remote administration manage queue manage documents spoolss remote manage

## Q111966: WFWG WinPopup Incorrectly Reports Print Job Completed

	Article: Q111966
	Product(s): Microsoft Windows 3.x Retail Product
	Version(s): WINDOWS:3.11
	Operating System(s): 
	Keyword(s): 
	Last Modified: 23-OCT-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Windows for Workgroups version 3.11 
	-------------------------------------------------------------------------------
	
	SYMPTOMS
	========
	
	WinPopup incorrectly reports to the client workstation that a print job has been
	completed if a Windows for Workgroups 3.11 network print job is canceled at the
	workstation that is sharing the printer.
	
	STATUS
	======
	
	Currently, there is no known workaround or resolution for this issue.
	
	Microsoft has confirmed this to be a problem in Windows for Workgroups version
	3.11. We are researching this problem and will post new information here in the
	Microsoft Knowledge Base as it becomes available.

## Q165371: XADM: Error Creating NT Account in Exchange Administrator

	Article: Q165371
	Product(s): Microsoft Exchange
	Version(s): 5.0
	Operating System(s): 
	Keyword(s): kbusage
	Last Modified: 07-MAY-1999
	
	-------------------------------------------------------------------------------
	The information in this article applies to:
	
	- Microsoft Exchange Server, version 5.0 
	-------------------------------------------------------------------------------
	
	
	SYMPTOMS
	========
	
	When you create a Microsoft Exchange mailbox in the Microsoft Exchange Server
	Administrator program and you create a new Windows NT account, the following
	error message appears:
	
	  The error '317' occurred while formatting Windows NT error '2245'.
	  Microsoft Windows NT
	  ID no. 0xc00208c5
	
	This only occurs when a minimum password length is required.
	
	STATUS
	======
	
	Microsoft has confirmed this to be a problem in Microsoft Exchange Server
	version 5.0 This problem was corrected in the latest Microsoft Exchange Server
	5.0 U.S. Service Pack. For information on obtaining the service pack, query on
	the following word in the Microsoft Knowledge Base (without the spaces):
	
	  S E R V P A C K

DOCUMENT:Q163460  14-NOV-2000  [word97]
TITLE   :WD97: ODMA TeamLinks or PCDOCS: Images Missing Saving to HTML
PRODUCT :Word 97 for Windows
PROD/VER:WINDOWS:97
OPER/SYS:
KEYWORDS:kbinterop

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------


SYMPTOMS
========

When you save Word documents containing images to HTML that are managed by
either ODMA TeamLinks or PCDOCS, the following error message appears:

   Additional files for each embedded picture or object will be saved in
   '<path>'. They will be named image*.*. For this document to display
   correctly, these files should be copied or moved to the document's location.

STATUS
======

Microsoft has confirmed this to be a problem in the products listed at the
beginning of this article.


MORE INFORMATION
================

Copying or moving the image files to the path specified in the error message
doesn't allow for the images to appear in the HTML document.

For additional information, please see the following article in the Microsoft
Knowledge Base:

   Q159963 Word 97 Supports ODMA

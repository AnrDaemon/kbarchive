TITLE   :Problems Printing From DCA IRMA Workstation for NT 1.1
PRODUCT :Microsoft Windows NT
PROD/VER:
OPER/SYS:
KEYWORDS:

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation version 3.51 
 - Microsoft Windows NT Server version 3.51 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you run DCA IRMA Workstation for NT 1.1 and then select File, Print, and
Screen Print, the following error appears and DCA IRMA terminates:

   Standard: EMU.EXE -Application Error
   The instruction at 0x77f76eb9 referenced memory at 0x00000008. The
   memory could not be "read".

This problem does not occur under Windows NT 3.5, nor does it occur with earlier
versions of DCA IRMA Workstation.

RESOLUTION
==========

To correct this problem, install the latest U.S. Service Pack for Windows NT
version 3.51.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This
problem has been corrected in the latest U.S. Service Pack for Windows NT
version 3.51. For information on obtaining the Service Pack, query on the
following word in the Microsoft Knowledge Base (without the spaces):

   S E R V P A C K

DCA IRMA Workstation for NT is manufactured by DCA, a vendor independent of
Microsoft; we make no warranty, implied or otherwise, regarding this product's
performance or reliability.

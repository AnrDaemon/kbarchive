DOCUMENT:Q117790  08-AUG-2001  [winnt]
TITLE   :Chooser Fails on Macintosh SE Running System 7.1
PRODUCT :Microsoft Windows NT
PROD/VER:3.1 3.5 3.51 4.0
OPER/SYS:
KEYWORDS:kbinterop

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Advanced Server, version 3.1 
 - Microsoft Windows NT Server versions 3.5, 3.51, 4.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you have Services for Macintosh installed and you use Chooser to connect to
AppleShare and select a file server, the following message may appear:

   Address Error
   Restart

This message appears when you are using a Macintosh SE with System 7.1 that is
connected to a Microsoft UAM volume on Windows NT Server.

CAUSE
=====

On Macintosh SE computers running System 7.1, you cannot select a file server
from the Chooser when the AppleShare folder is in the Macintosh System folder.


STATUS
======

There is no workaround for this problem at this time. For more information, call
Apple Technical Support at (800) 776-2333.

DOCUMENT:Q245352  11-DEC-2001  [winnt]
TITLE   :Administrators Are Unable to Add Either Local or Domain Printers
PRODUCT :Microsoft Windows NT
PROD/VER:winnt:4.0,4.0 SP4,4.0 SP5
OPER/SYS:
KEYWORDS:kberrmsg kbprint

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Server versions 4.0, 4.0 SP4, 4.0 SP5, Terminal Server Edition 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you try to install a printer, you may receive a request for the appropriate
printer driver files. After you click OK, you receive a message prompting you to
provide the path to the needed information (.inf) files for that printer. When
you try to provide the requested files from CD-ROM or a network share, you may
be unable to do so.

RESOLUTION
==========

To resolve this problem, obtain the latest service pack for Windows NT Server
version 4.0, Terminal Server Edition. For additional information, please see the
following article in the Microsoft Knowledge Base:

   Q152734 How to Obtain the Latest Windows NT 4.0 Service Pack

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products that are
listed at the beginning of this article. This problem was first corrected in
Windows NT Server version 4.0, Terminal Server Edition Service Pack 5.

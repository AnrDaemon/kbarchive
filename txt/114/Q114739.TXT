TITLE   :AppleTalk Print Jobs Stay in Queue and Don't Print
PRODUCT :Microsoft Windows NT
PROD/VER:3.1
OPER/SYS:
KEYWORDS:kbprint kbPrinting

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Advanced Server, version 3.1 
-------------------------------------------------------------------------------

SYMPTOMS
========

Printers connected through AppleTalk may hang in the queue and won't print if
the printer runs out of paper during a print job.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT and Windows NT
Advanced Server version 3.1. This problem has been corrected in the latest U.S.
Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For
information on obtaining the Service Pack, query on the following word in the
Microsoft Knowledge Base (without the spaces):

   S E R V P A C K

The AppleTalk product included here is manufactured by Apple Computer, Inc., a
vendor independent of Microsoft; we make no warranty, implied or otherwise,
regarding this product's performance or reliability.

DOCUMENT:Q292288  11-DEC-2001  [winnt]
TITLE   :Terminal Server Users Lose Drive Mappings
PRODUCT :Microsoft Windows NT
PROD/VER::4.0
OPER/SYS:
KEYWORDS:kb3rdparty kbnetwork

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Server version 4.0, Terminal Server Edition 
-------------------------------------------------------------------------------

SYMPTOMS
========

When client computers connect to a Terminal server that is participating in a
Novell network and that is using the Novell client version 4.7, 4.71 or 4.8.,
clients inherit the drive mapping of the last logged on user, which may seem as
if the clients are losing their drive mappings. In addition, when the last
logged on user logs off, all drive mappings for all clients disappear.

CAUSE
=====

This problem is caused by the Novell client. More details can be found in the
Novell Knowledge Base Article 10051948, which is located at the following Novell
Web site:

   support.novell.com

RESOLUTION
==========

You can obtain a fix from Novell Technical Support by going to the following
Novell Web site:

   support.novell.com

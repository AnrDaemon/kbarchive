The information in this article applies to:

 - Microsoft Mail for AppleTalk Networks, version 3.1a 
-------------------------------------------------------------------------------

SUMMARY
=======

Version 3.1a of Microsoft Mail for AppleTalk Networks maintenance release
contains the following corrections and enhancements:

 - Mail can no longer be delivered in one direction when using Server Connection
   with large mail installations.

   Intelligent handling of aliases:

    - Aliases can now be placed in MS Remote Sites folder.

    - Aliases are handled correctly when utilizing the enclosure screen; The Add
      button correctly adds the alias, the open button correctly resolves the
      alias.

 - The Remote Server Connection application will no longer pause causing CPU
   slowdown.

 - Increased listing of available CPU types for the Mail Network Admin.

 - PowerBooks are allowed to spin down the hard drive and sleep as configured.

 - The first user or group in the local listing can not be arbitrarily deleted
   by the system.

 - MS Mail GW correctly reconnects to the selected server upon restart when
   running Microsoft Mail as an application.

 - HyperCard API Move/Copy Returns correct destination folder ID.

 - Enhancement of recovery for local data files.

 - Message summary screen will always be drawn on the screen.

 - The workstation installation program will not corrupt AppleTalk resources
   when using System 7.1.

MORE INFORMATION
================

Mail 3.1a also included a new Move User feature enhancement of the Mail Network
Admin program. This feature allows administrators to extract users from one Mail
server and insert them into another. This provides an easy approach for moving
users from server-to-server while preserving mail and folder structures.

The version 3.1 Microsoft Mail upgrade can be ordered from Microsoft Inside Sales
at (800) 227-4679. International customers can request the upgrade through their
local subsidiary.

TITLE   :Err Msg: "Access denied" Running MS-Mail SMTP Gateway in VDM
PRODUCT :Microsoft Windows NT
PROD/VER::3.5
OPER/SYS:
KEYWORDS:

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation version 3.5 
 - Microsoft Windows NT Server version 3.5 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you run Mail Gateway to SMTP for Windows NT at a Windows NT command prompt,
an "Access denied" error message appears when mail is sent to or received from
the gateway server.

NOTE: Mail Gateway to SMTP for Windows NT is not an official Microsoft product,
and is not supported by Microsoft Product Support Services (PSS).

Mail Gateway to SMTP for Windows NT was ported to the Windows NT platform by an
intern. It was never released as a product; it was simply placed on the
Internet. PSS was not involved in the port or release of Mail Gateway to SMTP
for Windows NT and does not support it.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT version 3.5. This
problem was corrected in the latest U.S. Service Pack for Windows NT version
3.5. For information on obtaining the Service Pack, query on the following word
in the Microsoft Knowledge Base (without the spaces):

S E R V P A C K

Additional query words: prodnt 3.50 Mail 3.02

DOCUMENT:Q147601  08-AUG-2001  [winnt]
TITLE   :Problem Switching From Office 95 to 16-bit Application Window
PRODUCT :Microsoft Windows NT
PROD/VER:winnt:3.51
OPER/SYS:
KEYWORDS:kbinterop

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation version 3.51 
 - Microsoft Windows NT Server version 3.51 
-------------------------------------------------------------------------------


SYMPTOMS
========

If the window of an Office 95 application covers or overlaps the window of a
16-bit application and you switch to the 16-bit application, the window does not
appear correctly. The user interface of the Office 95 application that covered
the 16-bit application window appears when you switch to the 16-bit
application.



RESOLUTION
==========

To work around the problem, resize or minimize the 16-bit application window and
restore it.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This
problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For
information on obtaining the Service Pack, query on the following word in the
Microsoft Knowledge Base (without the spaces):

   S E R V P A C K


Additional query words: prodnt repaint screen display

TITLE   :Cyrillic Characters Are Not Displayed Properly in Some Programs
PRODUCT :Microsoft Windows NT
PROD/VER:WinNT:4.0
OPER/SYS:
KEYWORDS:kbenv

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Server version 4.0 
 - Microsoft Windows NT Workstation version 4.0 
 - Microsoft Windows NT Server, Enterprise Edition version 4.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

Cyrillic characters may not be displayed correctly in programs, such as
HyperTerminal, when you are using the Terminal font.

Note that the settings you choose on the Regional Settings tab in the Regional
Settings tool in Control Panel do not affect this symptom.

CAUSE
=====

Cyrillic characters are not displayed correctly when you use the Terminal font.

RESOLUTION
==========

To resolve this issue, change the default font in the program you are using to
Courier New. For information about changing the default font in the program you
are using, consult the program's documentation or manufacturer.


Additional query words: fonts character garbage unintelligible

TITLE   :XCLN: Deleted Items Folder Is Emptied When Installing a Form
PRODUCT :Microsoft Exchange
PROD/VER::; WINDOWS:
OPER/SYS:
KEYWORDS:

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Outlook 97 
 - Microsoft Outlook 98 
 - Microsoft Outlook 2000 
 - Microsoft Exchange Server, version 5.5 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you install forms that you create by using Microfot Exchange Forms Designer
in a public folder, Outlook folder, Personal Forms Library, or Organizational
Forms Library, your Deleted Items folder is emptied.

WORKAROUND
==========

If the Deleted Item Recovery feature is active, you can recover the deleted
items. For additional information about recovering deleted items, click the
article number below to view the article in the Microsoft Knowledge Base:

   Q228934 XCLN: Understanding Deleted Item Recovery


MORE INFORMATION
================

For additional information about installing forms that you create with EFD,
click the article number below to view the article in the Microsoft Knowledge
Base:

   Q146253 XCLN: How to Install a Form Created with EFD

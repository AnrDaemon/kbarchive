TITLE   :Conditions Where /PACKCODE is Not the Default
PRODUCT :Microsoft Programming Utilities
PROD/VER:MS-DOS:5.01,5.02,5.03,5.05,5.1,5.11,5.13,5.15,5.2,5.3
OPER/SYS:
KEYWORDS:kb16bitonly

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft LINK for MS-DOS, versions 5.01, 5.02, 5.03, 5.05, 5.1, 5.11, 5.13, 5.15, 5.2, 5.3 
-------------------------------------------------------------------------------

SUMMARY
=======

/PACKCODE is normally the default when linking. However, if either of the
following is true, /NOPACKCODE is the default:

1. Segments are defined in the .DEF file

-or-

2. The /INCR[EMENTAL] switch is given

Additional query words: kbinf 5.10 5.20 5.30

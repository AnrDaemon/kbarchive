SUMMARY
=======

This article contains comments, corrections, and information about known errors
relating to the Microsoft Press book Analyzing Requirements and Defining
Solution Architectures MCSD Training Kit, ISBN 0-7356-0854-7.

The following topics are covered:

 - Pages 84 and 664: Corrections For Question 5

 - Page 134: Text Correction

 - Page 275: Step 3 Should Be Implementation

 - Back Cover Refers To "System Requirements"

 - CD-ROM: Ebook Contains A Blank Section

MORE INFORMATION
================

Pages 84 and 664: Corrections For Question 5
--------------------------------------------

Page 84, change:
"Identify five guiding software management principles."

To:
"Identify five of the guiding software management principles."

On page 664, add the following items to the bulleted answer list:

 - Project success factors

 - Team approach

 - Individual commitment to project goals

 - Early product demonstrations

 - Risk management

 - Component-based development

 - Change management

 - Product versions based on customer priorities and expectations

 - Flexible, scalable framework


Page 134: Text Correction
-------------------------

Page 134, last paragraph, third sentence:

Change:
"For example, the inception stage may require four iterations through the five
RSIDM phases."

To:
"For example, the inception stage may require four iterations through the five
phases."


Page 275: Step 3 Should Be Implementation
-----------------------------------------

On page 275, the heading for step 3 should be Implementation.

Change:
"Step#3: Rationalization"

To:
"Step#3: Implementation"


Back Cover Refers To "System Requirements"
------------------------------------------

On the back cover the statement:
"Please see System Requirements in the section entitled "About This Book" for
information about using the CD-ROM"

Should read:
"Please see Getting Started in the setion entitled "About This Book" for
information about the CD-ROM."


CD-ROM: Ebook Contains A Blank Section
--------------------------------------

Under "Case Study, the RMS Project Team", the section titled "Case Study 02:
Introducing the RMS Project Team" is blank.

To resolve this issue, replace the ARDSA.HLP file that you copied from the CD-ROM
to your hard drive with the corrected ARDSA.HLP file. This corrected file is
available for download as part of ARDSA.EXE, a self-extracting zip file.

For information regarding installing and running this ebook, please refer to the
README.TXT file that can be found in the ebook directory of your CD-ROM.

The following file is available for download from the Microsoft Download Center:

   ARDSA.EXE
   (http://download.microsoft.com/download/MSPressPub/Patch/2.0/NT4/EN-US/ARDSA.EXE)

For additional information about how to download Microsoft Support files, click
the article number below to view the article in the Microsoft Knowledge Base:

   Q119591 How to Obtain Microsoft Support Files from Online Services

Microsoft used the most current virus detection software available on the date of
posting to scan this file for viruses. Once posted, the file is housed on secure
servers that prevent any unauthorized changes to the file.


Microsoft Press is committed to providing informative and accurate books. All
comments and corrections listed above are ready for inclusion in future
printings of this book. If you have a later printing of this book, it may
already contain most or all of the above corrections.

TITLE   :XCLN: Phonebook Entries w/ Extended Chars Not Sorted Correctly
PRODUCT :Microsoft Exchange
PROD/VER:4.0 5.0
OPER/SYS:
KEYWORDS:kbinterop kbnetwork

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Exchange Windows 3.x client, versions 4.0, 5.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

When using the 16-bit Microsoft Exchange client running on Microsoft Windows for
Workgroups version 3.x, phone book entries used with Remote Mail may not be
sorted correctly if they contain extended characters.

WORKAROUND
==========

You can work around this problem by selecting International in the Windows for
Workgroups Control Panel and changing the language setting from English
(American) to English (International).

STATUS
======

Microsoft has confirmed this to be a problem with the 16-bit Exchange client
running on Microsoft Windows for Workgroups. We are researching this problem and
will post new information here in the Microsoft Knowledge Base as it becomes
available.

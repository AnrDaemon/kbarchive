TITLE   :Improper Form Feed with Diconix 150 Plus Printer
PRODUCT :Microsoft Windows NT
PROD/VER:winnt:3.5,3.51
OPER/SYS:
KEYWORDS:kbprint kbPrinting

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation versions 3.5, 3.51 
 - Microsoft Windows NT Server versions 3.5, 3.51 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you define 4 inch high custom form and then print to a Diconix 150 Plus
driver from any application using that form, the printer uses a form feed at the
end of the document as if you had used an 8.5 x 11 inch form.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT version 3.51. A
supported fix is now available, but is not fully regression-tested and should be
applied only to systems experiencing this specific problem. Unless you are
severely impacted by this specific problem, Microsoft recommends that you wait
for the service pack release containing this fix. Contact Microsoft Product
Support Services for more information.

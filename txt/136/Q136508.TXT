TITLE   :Filtered Events For Server Tools Event Viewer Not Saved
PRODUCT :Microsoft Windows NT
PROD/VER::3.5,3.51
OPER/SYS:
KEYWORDS:

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation versions 3.5, 3.51 
 - Microsoft Windows NT Server versions 3.5, 3.51 
-------------------------------------------------------------------------------


SYMPTOMS
========

After you filter events in Windows NT Server Tools Event Viewer, enable Save
Settings on Exit, exit Event Viewer, and restart Event Viewer, the default set
of events are displayed. If you select Filter Events in the View menu, all
filter event settings are set back to the original default settings.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT versions 3.5 and
3.51. We are researching this problem and will post new information here in the
Microsoft Knowledge Base as it becomes available.

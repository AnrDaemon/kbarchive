DOCUMENT:Q148596  13-JUN-2001  [sna]
TITLE   :Event ID 951 Should Not Be Logged As A Significant System
PRODUCT :Microsoft SNA Server
PROD/VER:WINDOWS:2.0,2.1,2.11,2.11 SP1
OPER/SYS:
KEYWORDS:kbnetwork

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft SNA Server, versions 2.0, 2.1, 2.11, 2.11 SP1, on platform(s):
    - the operating system: Microsoft Windows NT 
-------------------------------------------------------------------------------


SYMPTOMS
========

The following is logged as an application event in Event Viewer even though the
Default Audit Log Level has been set to Significant System Events:

   Event ID: 951
   Source: SNA Base Service
   Type: Warning
   Description: No application accepted received message.

CAUSE
=====

Event ID 951 is generated by the SNA Server client. This events occurs when a
3270 emulator does not accept a message passed by the Windows 3.x SNA Server
client. The event contains no significant information that may be useful for
troubleshooting purposes.

RESOLUTION
==========

To correct this problem, apply the fix mentioned below.

STATUS
======

Microsoft has confirmed this to be a problem in SNA Server version 2.11. This
problem was corrected in the latest Microsoft SNA Server 2.11 U.S. Service Pack.
For information on obtaining the service pack, query on the following word in
the Microsoft Knowledge Base (without the spaces):

   S E R V P A C K


Additional query words: prodsna

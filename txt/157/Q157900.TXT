DOCUMENT:Q157900  09-APR-1999  [exchange]
TITLE   :XCLN: Incorrect Characters Displayed in Profile for NT Client
PRODUCT :Microsoft Exchange
PROD/VER:WINDOWS:4.0
OPER/SYS:
KEYWORDS:kbusage

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Exchange Windows NT client, version 4.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you attempt to create a new profile using the Chinese version of the
Microsoft Exchange client for Windows NT, the Name edit line displayed on the
screen will not be the same as what you entered with the keyboard.

CAUSE
=====

This is because the default font for Windows NT version 4.0 is incorrect.

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft Exchange clients,
version 4.0, listed at the beginning of this article. This problem was corrected
in Microsoft Exchange Server 4.0 U.S. Service Pack 4. For information on
obtaining the service pack, query on the following word in the Microsoft
Knowledge Base (without the spaces):

   S E R V P A C K

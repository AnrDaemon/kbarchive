TITLE   :XCOPY Fails with InocuLAN Virus Software Installed
PRODUCT :Microsoft Windows NT
PROD/VER:3.1 3.5 3.51
OPER/SYS:
KEYWORDS:kb3rdparty

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Server version 3.1 
 - Microsoft Windows NT Workstation version 3.1 
 - Microsoft Windows NT Advanced Server, version 3.1 
 - Microsoft Windows NT Workstation versions 3.5, 3.51 
 - Microsoft Windows NT Server versions 3.5, 3.51 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you attempt to do an XCOPY from a DOS 6.22 client with NETX/ODI to an empty
directory on a Windows NT computer running Cheyenne InocuLAN software, the XCOPY
stops responding.

RESOLUTION
==========

To resolve this problem please contact Cheyenne technical support at the
number(s) listed below to obtain the latest version of the InocuLAN NT software
for Windows NT 3.51 (version 1.01 build 51).

   Cheyenne
   USA Three Expressway Plaza
   Roslyn Heights, NY 11577
   Voice: (800) 243-9832
   Fax: (516) 465-5115
   Cheyenne BBS: (516)465-3900
   InnocuLAN BBS: (516)465-3930
   http://www.cheyenne.com

MORE INFORMATION
================

Steps to Reproduce the Problem
------------------------------

1. Run Cheyenne InocuLAN Windows NT service.

2. Direct the XCOPY to an empty destination subdirectory.

3. Use NETX/ODI DOS6.22 client to XCOPY files to File and Print Services for
   NetWare (FPNW) server.

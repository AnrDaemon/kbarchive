DOCUMENT:Q150831  08-AUG-2001  [winnt]
TITLE   :Stop 0x0000000A in NBF.SYS When Running Under Stress
PRODUCT :Microsoft Windows NT
PROD/VER:winnt:3.51
OPER/SYS:
KEYWORDS:kbnetwork

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Windows NT Workstation version 3.51 
 - Microsoft Windows NT Server version 3.51 
-------------------------------------------------------------------------------


SYMPTOMS
========

When a high number of clients connect to your Windows NT computer using the
NetBEUI protocol (Nbf), a STOP message appears with a Trap 0x0000000A
IRQL_NOT_LESS_OR_EQUAL.


STATUS
======

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This
problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For
information on obtaining the Service Pack, query on the following word in the
Microsoft Knowledge Base (without the spaces):

   S E R V P A C K


Additional query words: blue trap nbf

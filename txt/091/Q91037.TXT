TITLE   :PC WFW: Unable to Address External Mail Users
PRODUCT :Microsoft Mail For PC Networks
PROD/VER:WINDOWS:3.0
OPER/SYS:
KEYWORDS:

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Mail for PC Networks, version 3.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

After you use version 3.0 of Extensions for Windows for Workgroups to upgrade a
Windows version 3.1 for Workgroups postoffice to a version 3.0 Microsoft Mail
for PC Networks postoffice, users are unable to address mail to external users.

CAUSE
=====

You must grant external rights to the users that need to mail external users.

RESOLUTION
==========

To grant external mail privileges to a user:

1. Run the Mail Administrator program (ADMIN.EXE).

2. From the Local-Admin menu, choose Modify.

STATUS
======

Microsoft has confirmed this to be a problem in version 3.0 of Extensions for
Windows for Workgroups. We are researching this problem and will post new
information here in the Microsoft Knowledge Base as it becomes available.

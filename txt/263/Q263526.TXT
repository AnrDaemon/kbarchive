DOCUMENT:Q263526  10-NOV-2000  [exchange]
TITLE   :XADM: Appointments Sent in Different Time Zones Not Processed
PRODUCT :Microsoft Exchange
PROD/VER:winnt:5.5
OPER/SYS:
KEYWORDS:exc55 kbExchange550preSP4fix kbExchange550sp4Fix

======================================================================
-------------------------------------------------------------------------------
The information in this article applies to:

 - Microsoft Exchange Server, version 5.5 
-------------------------------------------------------------------------------


SYMPTOMS
========

If the difference between the sending and accepting time zones is more than 12
hours, Collaboration Data Objects (CDO) may not process meeting requests
correctly.

CAUSE
=====

This issue can occur because of a logic error in the initialization of the time
zone variable in CDO code.

RESOLUTION
==========

To resolve this problem, obtain the latest service pack for Exchange Server 5.5.
For additional information, please see the following article in the Microsoft
Knowledge Base:

   Q191014 XGEN: How to Obtain the latest Exchange Server 5.5 Service Pack



STATUS
======

Microsoft has confirmed this to be a problem in Microsoft Exchange Server
version 5.5. This problem was first corrected in Exchange Server 5.5 Service
Pack 4.

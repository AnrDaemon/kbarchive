Q129084: WRITE.EXE Does Not Save the Current Font Setting

Article: Q129084
Product(s): Microsoft Windows 95.x Retail Product
Version(s): WINDOWS:3.1,3.11
Operating System(s): 
Keyword(s): 
Last Modified: 01-NOV-1999

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows versions 3.1, 3.11 
- Microsoft Windows for Workgroups versions 3.1, 3.11 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you save a document in Microsoft Write, if the cursor is at the end-of-
file marker and the font is something other than Arial, the current font setting
is not saved.

RESOLUTION
==========

To save the current font setting, use one of the following methods:

- Insert spaces or press the LEFT ARROW key before you save the document.

- Press the LEFT ARROW key and then press the RIGHT ARROW key after you save
  the document.

- Open the document after you save it. Note that this method prevents the
  problem from re-occurring. The problem occurs only during the initial
  session.

STATUS
======

Microsoft is researching this problem and will post new information here in the
Microsoft Knowledge Base as it becomes available.

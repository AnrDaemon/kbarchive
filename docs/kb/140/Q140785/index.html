Q140785: PPT7: Style Checker Capitalizes Words Incorrectly

Article: Q140785
Product(s): Microsoft PowerPoint for Windows
Version(s): WINDOWS:7.0
Operating System(s): 
Keyword(s): kbusage kbdtakbbuglist
Last Modified: 16-APR-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft PowerPoint for Windows 95, version 7.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

If you run Style Checker, some of your text may be capitalized incorrectly.

CAUSE
=====

This problem occurs when text that the Style Checker changes to Title Case
contains an apostrophe.

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products listed at
the beginning of this article.

RESOLUTION
==========

1. On the Tools menu, click Style Checker. Click OK to start the Style Checker.

2. Once Style Checker is complete, start it again. Under Check For, clear the
  Case, End Punctuation, and Visual Clarity check boxes. Leave only the
  Spelling check box selected.

The Spell Checker will find any punctuation that is not correct and ask you if
you want to change it.

MORE INFORMATION
================

Q151960: FP: Table Column Widths Don’t Equal Total Width of Table

Article: Q151960
Product(s): Word Front Page
Version(s): WINDOWS:1.1
Operating System(s): 
Keyword(s): kbdta
Last Modified: 04-OCT-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft FrontPage 97 for Windows with Bonus Pack 
- Microsoft FrontPage for Windows 1.1 
-------------------------------------------------------------------------------

For a Microsoft FrontPage 98 version of this article, see Q194468.

SUMMARY
=======

In FrontPage Editor, it is possible to create a table such that the total widths
of all columns do not equal the total width of the table.

When a table is created with these characteristics, the appearance of the table
will vary depending on the browser you use to view it.

MORE INFORMATION
================

For best results, if you create a table using columns of fixed percentage
widths, make sure the total of the column widths is less than or equal to the
total width of the table.

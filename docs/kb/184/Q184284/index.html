Q184284: WD97: Cannot Find Grammar DLL During Spell Check

Article: Q184284
Product(s): Word 97 for Windows
Version(s): WINDOWS:
Operating System(s): 
Keyword(s): kb3rdparty kberrmsg kbdta kbproof word97
Last Modified: 14-NOV-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------

SYMPTOMS
========

You may receive the following error message when using Alki(R) Language Proofing
Tools to check the spelling in a Word document:

  Cannot find msgr**_32.dll

Where ** is a two-letter abbreviation designating the language you are proofing.

CAUSE
=====

This problem may occur after you upgrade to Word 97 for Windows and you do not
upgrade your proofing tools to the Alki Language Proofing Tools for Office 97
for Windows.

WORKAROUND
==========

To work around this problem, turn off the "Check Grammar with Spelling" option.
To do this, follow these steps:

1. On the Tools menu, click Options.

2. On the Spelling & Grammar tab, click to clear both the "Check grammar
  with spelling" and "Check grammar as you type" check boxes (under Grammar).

MORE INFORMATION
================

For more information, please see the following article in the Microsoft
Knowledge Base:

  Q186037 OFF97: Alki Language Proofing Tools Available
